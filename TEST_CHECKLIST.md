## Step 4: 툴팁 정합성 검증 (★자동화 필요★)

**원칙: Financials 본체 값 = Frontdata 하위항목 합계**

현재 수동 확인만 가능. API 응답의 `vcm`(Frontdata)과 `vcm_display`(Financials)를 프로그래밍으로 비교하는 자동화 검증이 필요.

| # | 검증 항목 | 설명 | 자동화 |
|---|----------|------|--------|
| 1 | **기타유동자산** | 본체 = 툴팁 하위 합계 | 가능 |
| 2 | **기타비유동자산** | 본체 = 툴팁 하위 합계 | 가능 |
| 3 | **기타유동부채** | 본체 = 툴팁 하위 합계 | 가능 |
| 4 | **기타비유동부채** | 본체 = 툴팁 하위 합계 | 가능 |
| 5 | **기타자본구성요소** | 본체 = 자본잉여금 + 자본조정 + 기타포괄손익누계액 등 | 가능 |
| 6 | **NWC** | NWC = 유동자산 - 유동부채 | 가능 |
| 7 | **Net Debt** | Net Debt = 유동차입부채 + 비유동차입부채 - 현금 - 단기투자 | 가능 |
| 8 | **EBITDA** | EBITDA = 영업이익 + 감가상각비 + 무형자산상각비 | 가능 |
| 9 | **이름 중복 없음** | vcm 전체에서 동일 항목명 2개 이상 없어야 함 | 가능 |
| 10 | **Financials 필터링** | overflow 하위항목은 Financials에서 제외, Frontdata에만 존재 | 가능 |

**자동화 방법:** `vcm` 응답에서 `부모` 필드가 있는 항목의 합계와, `vcm_display`의 해당 본체 값을 비교. 백만원 변환 후 ±1 이내 일치 확인.

## Step 5: 엑셀-프론트 데이터 일치 검증 (★핵심★)

**원칙: 엑셀 Financials 시트 숫자 = 웹 UI에 표시되는 숫자 (1:1 일치)**

| # | 검증 항목 | 설명 |
|---|----------|------|
| 1 | **엑셀 다운로드** | `GET /api/download/{task_id}` → 파일 수신, 크기 > 0 |
| 2 | **필수 시트 존재** | `Financials`, `Frontdata` 시트 존재 |
| 3 | **Financials 행 수** | 10행 이상 데이터 존재 |
| 4 | **Frontdata 행 수** | 10행 이상 데이터 존재 |
| 5 | **Financials = vcm_display** | 엑셀 Financials 시트의 각 셀 수치 = API `vcm_display` 각 셀 수치 (백만원 단위) |
| 6 | **Frontdata = vcm** | 엑셀 Frontdata 시트의 각 셀 수치 = API `vcm` 각 셀 수치 (원 단위) |
| 7 | **부모-자식 정합성** | Frontdata 하위 항목의 `부모` 필드가 유효한 항목을 참조 |
| 8 | **BS/IS/CF 시트 존재** | 재무상태표, 손익계산서, 현금흐름표 원본 시트 존재 |
| 9 | **단위 일관성** | Financials=백만원 정수, Frontdata=원 단위 |

**검증 방법:**
1. 엑셀 Financials 시트 열기
2. 웹 UI Financials 탭과 항목별 수치 1:1 비교
3. 특히 음수 값, 큰 숫자, 천 단위 콤마 확인
4. Frontdata의 하위 항목이 본체 항목에 올바르게 매핑되어 있는지 확인

## Step 7: BS Overflow 검증

BS 각 섹션에서 항목이 MAX_ITEMS=6개 초과 시 "기타{섹션명}"으로 합산되는 로직 검증.

| # | 검증 항목 | 설명 |
|---|----------|------|
| 1 | **섹션별 합계 일치** | 각 섹션 헤더값 = 직접 하위항목 합계 |
| 2 | **이름 중복 없음** | vcm 전체에서 동일 항목명 2개 이상 없어야 함 |
| 3 | **기타 컨테이너 정합성** | 기타{섹션} 값 = 하위항목 합계 |
| 4 | **이름 충돌 방지** | (개별)/(합산)/[섹션명] 접미사 올바르게 적용 |
| 5 | **다중 연도 일관성** | 모든 FY에서 동일한 항목이 selected/overflow |

### 이름 충돌 시나리오

| 시나리오 | 조건 | 기대 결과 |
|----------|------|----------|
| overflow 자식-컨테이너 충돌 | DART "기타비유동자산"이 overflow에 포함 | 자식 → "(개별)" 접미사 |
| selected-컨테이너 충돌 | DART "기타비유동자산"이 selected에 포함 | 컨테이너 → "(합산)" 접미사 |
| 다른 섹션 동명 항목 | "보증금"이 유동+비유동 양쪽 | [섹션명] 접미사 자동 |
